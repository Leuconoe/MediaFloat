# MediaFloat v0.1.0

MediaFloat v0.1.0 is the first public release of the project.

This version focuses on a compact Android overlay for core media controls, plus the setup and debugging surfaces needed to make that runtime understandable.

## Highlights

- Floating media controls for `Previous`, `Play / pause`, and `Next`
- Draggable overlay bar with a fixed right-side handle
- Settings screen with supported button layouts, size presets, and a live preview
- Debug console with runtime readiness, media readiness, transport controls, and recent logs
- Support screen with setup guidance, version details, and current constraints
- Exported automation action for launching the overlay flow from routines or shortcuts

## Setup notes

MediaFloat depends on Android overlay access, notification-listener access, and a visible foreground-service notification posture. If one of those pieces is missing, the app stays in a readable blocked state and points back to the system settings it needs.

## Known limitations

- The overlay supports a single horizontal control family in this release
- Freeform resizing and freeform button placement are not supported
- Validation has been strong on emulator-based setup and runtime checks, but target-device testing is still recommended before wider use

## Feedback focus

The most useful early feedback for `v0.1.0` is:

- media-app compatibility
- overlay behavior across OEM builds
- permission-flow clarity
- practical issues with positioning, persistence, and recovery
